Claude Code Tutorial

A beginner's guide for the Conservative AI Policy Fellowship

What is Claude Code?

Claude Code is a command-line tool that lets you work with Claude AI directly in your terminal. You describe what you want to build or fix, and Claude writes the code, creates files, and runs commands for you. Think of it as having a coding assistant that can actually do things on your computer, not just give advice.

You don't need to know how to code to use it. You just need to be able to describe what you want.

Installation Guide

Select your operating system, then follow the steps below. If you get stuck, don't worry—that's normal! Just describe the error to Claude once you're set up.

1

Open Terminal

Press Command + Space to open Spotlight, type Terminal, and press Enter.

Terminal
user@mac ~ %

This is the Terminal—where you'll type commands.

2

Install Claude Code

Copy and paste this command into Terminal, then press Enter:

curl -fsSL https://claude.ai/install.sh | bash

Wait for it to finish. You'll see some text scroll by—that's normal. When you see your cursor blinking on a new line, it's done.

After installation, close Terminal completely (Command + Q) and reopen it so the changes take effect.

3

Start Claude Code

Navigate to a folder where you want to work (or create a new one), then type:

claude

The first time you run this, it will ask you to log in with your Anthropic account. Follow the prompts in your browser.

Tip: Creating a project folder If you want to create a new folder for your project, use these commands:
mkdir my-project
cd my-project
claude
This creates a folder called "my-project," moves into it, and starts Claude.

Your First Command

Once Claude Code is running, you'll see a prompt where you can type. Try something simple:

Create a file called hello.txt that says "Hello from Claude Code!"

Claude will create the file for you. You can then open Finder (Mac) or File Explorer (Windows) and navigate to your project folder to see it.

Here are a few more things to try:

  • "What files are in this folder?" — Claude will look around and tell you.
  • "Create a simple HTML page that says Welcome" — Claude will write the code and save it.
  • "Explain what this project does" — If you're in an existing codebase, Claude will analyze it.

Starter Project Ideas

If you don't immediately think of a software-shaped problem in your life, I recommend picking a dead-simple starter project just to try it out. Here are some ideas for things to ask Claude:

  • Turn a CSV into a report with charts, plots, and insights. You could use some actual data you have, or try exporting your Goodreads data, Substack data, etc instead. Suggest edits, then save the report as a PDF.
  • Make a static website. This can be a personal website, an informational page, a gallery of images, or something else. For example, you can draft your website content in Google Docs, export it as a Markdown file (easy for Claude to read), and then tell Claude to convert it into a site. Then, play with the design by giving it inspiration images or fonts/colors/vibes. Finally, ask it to help you host it on a free Github account.
  • Build a personal tracker app. Tracker apps are simple and fun to personalize: Claude can make a site for you to log your books, movies, recipes, health stats, journaling, or anything else. Then actually use it, and fix bugs / add features as you go.
  • Automate an existing workflow. Think of a rote task you have to do, e.g. "get transcripts from YouTube videos" or "format a document according to brand guidelines." Explain to Claude what you do manually today, and ask it to write a script to help you automate it for the future. Test and debug until it works.
  • Design a custom game for a loved one. Claude can help you make a simple game like 20 Questions, Wordle, or Brick Breaker (or any other idea). What's fun is customizing the graphics and theme based on someone's unique interests, then building a site to send to them.

Tips for Beginners

  • Be specific about what you want. Instead of "make it better," say "make the heading larger and add more space between paragraphs."
  • It's okay to not understand the code. You can ask Claude "explain what you just did" or "why did you do it that way?" at any time.
  • Iterate and refine. Your first result won't be perfect. That's fine! Just tell Claude what to change: "make the background blue" or "add a button that does X."
  • If something breaks, describe the problem. Copy and paste any error messages you see. Claude is good at debugging.
  • Save your work with Git. Ask Claude to "initialize a git repository and commit my changes." This creates checkpoints you can return to if something goes wrong.
  • Take breaks. If you're frustrated, step away. The code will still be there when you get back.
Useful commands to know
  • /help — See all available commands in Claude Code
  • /clear — Clear the conversation and start fresh
  • Ctrl + C — Stop Claude if it's running too long
  • exit — Quit Claude Code and return to normal Terminal

Getting Help
